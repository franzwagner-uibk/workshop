Before you start

Bring these things and keep setup light

You do not need to finish the technical setup in advance. We can do VS Code, Git, and GitHub together during the workshop. The main thing to prepare beforehand is your content.

Bring this to the workshop

  1. Bring your laptop and charger.
  2. You do not need to install anything before the workshop.
  3. If you already have VS Code, Git, or a GitHub account, that is fine, but it is not required in advance.
  4. Keep your text, images, and links in one folder so they are easy to access during the workshop.

Please prepare your content before the workshop

  • Your name or artist name
  • One short introduction, 3 to 5 sentences
  • Project titles and short descriptions
  • Links: Instagram, email, website, CV, event page
  • 2 to 8 images you are allowed to publish

The goal

Keep it simple. Finish something real.

This workshop is for beginners

You do not need coding experience. We will use plain HTML, CSS, and a little JavaScript. We will also use the AI chat tool available in your VS Code to help us write, edit, and improve the website.

A good first project

Build one small site with clear content: your name, a short bio, one image, contact information, and 3 to 6 projects or references.

Done is better than perfect

Your first version can be simple. A clean one-page website is a great result. You can always improve it later.

Publish early, then improve

Try to get a very small version online early in the workshop. After that, use local preview in VS Code for fast changes and push new updates at a few checkpoints.

Basics

What GitHub, VS Code, and coding agents are

What is GitHub?

GitHub is a website where you store your project files online. It keeps a history of your changes, makes sharing easier, and is also where you can publish your site with GitHub Pages.

What is VS Code?

VS Code is the editor where you open your website folder and work on your files. You can write HTML, CSS, and JavaScript, preview changes locally, and use built-in Git tools to commit and push your work.

What are coding agents?

Coding agents are AI tools that can help you plan, write, explain, and fix code. They are useful for small steps and quick questions, but you should still read the changes, test them, and ask for explanations when something is unclear.

Workshop flow

Follow these steps in order

Step 1

Choose one clear website idea

Pick a small first version. Examples: personal homepage, portfolio, artist page, event page, studio page, or project archive.

Ask your AI chat in VS Code:

Help me plan a simple one-page website for my [portfolio / project / event]. I am a beginner. Suggest 5 sections and keep the structure very simple.

Step 2

Create your project folder

In VS Code, create a new folder for your site. Add these files: index.html, styles.css, and script.js.

Starter file list

index.html
styles.css
script.js
images/

Step 3

Build a tiny first page with HTML

Start with the smallest possible version: your name, one short sentence, and one contact link. This is enough for a first live website.

Ask your AI chat in VS Code:

Create a very simple index.html for a one-page personal website. Start with only a headline, short intro, and contact link. Keep it extremely simple because I am a beginner.

Simple HTML skeleton

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Your Name</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <header>
      <h1>Your Name</h1>
      <p>One short sentence about your work.</p>
    </header>

    <main>
      <section>
        <h2>About</h2>
        <p>Write a short introduction here.</p>
      </section>

      <section>
        <h2>Projects</h2>
        <p>Add your projects here.</p>
      </section>

      <section>
        <h2>Contact</h2>
        <p>Add email or social links here.</p>
      </section>
    </main>

    <script src="script.js"></script>
  </body>
</html>

Step 4

Publish this first version to GitHub

Create a public repository on GitHub. In VS Code, publish your folder to that repository or use the Source Control panel to commit and push your files.

  1. Create a new public repository on GitHub.
  2. Commit your files in VS Code.
  3. Push them to GitHub.

This first push can be very small. The goal is to get online early, not to be finished.

Step 5

Turn on GitHub Pages

Open your repository on GitHub. Go to Settings → Pages. Under Build and deployment, choose Deploy from a branch, then select your main branch and the root folder.

After a short wait, your site will be live at a GitHub Pages URL, usually https://yourusername.github.io/your-repository-name/.

Step 6

Install Live Server for fast local preview

Install the Live Server extension for VS Code by Ritwick Dey. It adds a simple local web server and a Go Live button so you can preview changes immediately while you edit your website.

Important: this preview is only on your laptop. GitHub Pages is the public website.

Step 7

Add your real content and style

Now replace all placeholder text. Add your own words, project titles, descriptions, links, spacing, colors, and fonts. Keep sentences short and clear.

Ask your AI chat in VS Code:

Help me improve my website step by step. First add my real content, then write simple CSS to make it clean, readable, and mobile-friendly. Keep everything easy to understand because I am a beginner.

Step 8

Push updates at checkpoints

Push your work after important milestones: after adding the basic content, after adding images, and once more before the end of the workshop.

Ask your AI chat in VS Code:

Help me add a responsive image gallery or project section to my existing site. Keep it simple and tell me what to push to GitHub after I test it locally.

Prompting tips

How to work well with AI in VS Code

Good prompt habits

  • Say that you are a beginner.
  • Ask for one small change at a time.
  • Paste the code you want to change.
  • Ask the AI to explain what it changed.
  • Test locally first, then push when the change works.
  • Use words like simple, clean, clear, mobile-friendly.

Prompt examples

Useful prompts

I am a beginner. Please improve this HTML, but keep it simple.

Explain this CSS in plain English.

Make this layout work better on mobile screens.

Add a project card section with image, title, year, and short text.

Check my code for mistakes and tell me what to fix first.

Publishing

Final checklist for GitHub Pages

  1. Your homepage file is named index.html.
  2. Your repository is public.
  3. Your files are pushed to the default branch.
  4. GitHub Pages is enabled in repository settings.
  5. You preview locally with Go Live or another local server while editing.
  6. You wait 1 to 5 minutes after publishing.
  7. You test the live link on your phone.

When something breaks

Simple troubleshooting

The CSS does not load

Check the filename. It must match exactly: styles.css. Also check that the link in your HTML is correct.

The image does not appear

Check the image path, filename, and letter case. Put the image in the correct folder and make sure the extension matches.

Go Live works, but the public site does not change

This usually means you changed the local files but did not push the new version to GitHub yet. Local preview and GitHub Pages are not the same thing.

GitHub Pages is not live yet

Wait a few minutes, then refresh the page. If it still does not work, check Settings → Pages again.

You feel stuck

Go back one step. Ask the AI chat tool to explain the current code in simple words. Then make one small change, not five at once.

Build a simple first version. Improve it later.
