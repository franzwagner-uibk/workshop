# Workshop Website Instructions

Read `plan.md` before you make changes.

## Project goals

- Build a beginner-friendly static website.
- Keep the structure easy to understand and easy to edit later in VS Code.
- Use real HTML files that match real website pages.

## Allowed stack

- HTML
- CSS
- Small amounts of vanilla JavaScript when it is genuinely useful

## Do not use by default

- React
- Next.js
- Vue
- Svelte
- TypeScript
- Single-page app routing
- Component abstractions
- Build tools or bundlers

## File structure rules

- Use one real `.html` file per real page.
- Keep file names direct and readable, such as `index.html`, `about.html`, or
  `projects.html`.
- Keep CSS in `styles.css` unless the user clearly asks for something else.
- Keep JavaScript small and readable. If it is not needed, do not add it.

## Workflow rules

- Start by reading `plan.md`.
- If the plan is unclear, ask concise design and structure questions before
  writing code.
- Keep the first implementation small enough to publish early.
- Explain non-obvious file creation or structure changes in simple language.
- Prefer changes that a beginner can inspect and understand.

## Editing style

- Write semantic HTML with clear headings, sections, lists, links, and image
  markup.
- Keep navigation explicit and easy to follow.
- Preserve simple relative links between pages.
- Favor clarity over cleverness.
